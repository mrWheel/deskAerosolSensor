# ILI9341 on CYD (ESP32) - Final Working Notes

This file captures the latest proven display setup used in this repository.

## 1. Hardware profile

- Board class: ESP32 CYD style board with 2.8 inch panel
- Display controller: ILI9341
- SPI host: VSPI
- SPI and control pins:
  - SCLK = 14
  - MOSI = 13
  - MISO = 12
  - DC = 2
  - CS = 15
  - Backlight = 21

## 2. Required panel geometry and orientation

The panel works correctly with these settings:

- panel width = 320
- panel height = 240
- memory width = 320
- memory height = 240
- rotation = 4
- RGB order = true

Without explicit memory width and memory height, the panel can show shifted or wrapped output in some rotation combinations.

## 3. LovyanGFX panel configuration

Use this configuration pattern:

```cpp
auto config = panel.config();
config.pin_cs = 15;
config.pin_rst = -1;
config.pin_busy = -1;

config.panel_width = 320;
config.panel_height = 240;
config.memory_width = 320;
config.memory_height = 240;

config.offset_x = 0;
config.offset_y = 0;
config.offset_rotation = 0;

config.rgb_order = true;
config.invert = false;
```

And set:

```cpp
tft.setRotation(4);
```

## 4. SPI bus settings

Stable settings in this project:

```cpp
auto busCfg = bus.config();
busCfg.spi_host = VSPI_HOST;
busCfg.spi_mode = 0;
busCfg.freq_write = 40000000;
busCfg.freq_read = 16000000;
busCfg.spi_3wire = false;
busCfg.use_lock = true;
busCfg.dma_channel = SPI_DMA_CH_AUTO;
```

## 5. LVGL flush path that worked reliably

The final working flush path uses `pushImage`:

```cpp
tft.startWrite();
tft.pushImage(area->x1, area->y1, width, height, reinterpret_cast<lgfx::rgb565_t*>(colorMap));
tft.endWrite();
```

LVGL resolution is derived from the active display object (`tft.width()`, `tft.height()`).

## 6. Current environments

This repository intentionally keeps two environments:

- `cyd_ili9341`
  - Full dashboard build for ILI9341
  - Currently configured with `SIMULATION=1`
- `ili9341_r4`
  - Display-only test pattern mode (`TEST_DISPLAY=1`)

## 7. SIMULATION mode behavior

With `SIMULATION=1`:

- sensor reading is skipped
- all nine tiles update continuously
- values move slowly up and down
- each tile uses a different period and phase so peaks do not happen at the same time
- serial monitor prints periodic simulation snapshots

## 8. Quick display verification

In `ili9341_r4` test mode, verify:

- color bars are in logical order
- corner markers are in expected corners
- text is readable and not mirrored

## 9. Practical note on borders

A small inactive black edge can still be normal panel dead area near the FPC side and is not always removable in software.
